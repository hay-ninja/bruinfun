# AI Usage Statement

This project used GenAI as an assistive tool. We followed course policy that unmodified AI output is not counted as our work. Our work is the prompt intent, reflection, verification, and code decisions/changes we made.

## Syllabus Alignment

- AI use is allowed but must be explicitly marked when AI-generated code is used.
- For marked AI-generated code, we include the prompt context and a short reflection.
- Reflection includes why AI was used, what was accepted/changed/rejected, and how we verified quality.
- We do not treat raw AI output as final deliverable code.

## How AI Was Used In This Project

1. README and documentation support
- Helped draft and reorganize README sections, including UML explanations and project onboarding wording.

2. Design decision reflection
- Helped compare pros/cons of implementation patterns before final team decisions.

3. Intermittent security checks
- Used for occasional sanity checks on auth/session handling, endpoint validation, and hardening ideas.
- Team members reviewed and validated all resulting changes.

4. Merge conflict support
- Used to assist with stale-branch merge conflict cleanup.
- Final conflict resolutions and resulting code were selected and verified by team members.

## What Counts As Team Work

- Problem framing and prompt design
- Technical judgment and tradeoff decisions
- Manual code edits and integration work
- Testing, debugging, and verification
- Reflection writeups and attribution for AI-assisted sections

## Boundaries

- AI was not used as an autonomous replacement for engineering ownership.
- AI suggestions were reviewed before merge and only accepted when quality matched project standards.
- Final architecture, correctness, and release responsibility remained with the team.
